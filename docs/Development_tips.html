Ionic implements view caching which means: * The Angular controller is not constructed each time * The controller's $scope is detached from the angular lifecyle when it is cached and not active The main change is in Angular controllers you will put logic into the Ionic events, such as $ionicView.beforeEnter. You will see many examples in the codebase. Then when you want to to deploy your Cloud Code then from the /CloudCode folder you can simply run **parse deploy dev** **parse deploy prod** To update the version of the Parse SDK your Cloud Code uses then run (replacing 1.4.2 with the latest version) **parse jssdk 1.4.2** To update your Parse command line tool run **parse update** ## Development When updating an object it is better to pass in a change set to save() and not to modifiy the object first and then call save() so if the save fails then the object retains the original values **Don't do this** user.country = 'Australia' return user.save() **Do this** var changeSet = {country : 'Australia'} return user.save(changeSet) http://blog.parse.com/2015/03/25/chart-new-territory-with-parse-explorer/
